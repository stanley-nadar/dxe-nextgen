# Button Component

Button wrapper to ingest Contentful data into the existing Button molecule.

## Link

The link field is intended to be uniform across the board. For that reason, note: the link needs to be a full URL. ie:
`https://www.freshworks.com`

## Label Icon

If provided, the this field will override the label and render an icon only button.

## Start Icon

If provided, the this field will import an icon at the start of the button and will increase its length.

## End Icon

If provided, the this field will import an icon at the end of the button and will increase its length.

If Chevron Right is selected, on hover or focus, the button will have a transformation animation.

## Hierarchy

The hierarchy determines a button's priority level and determines its colors and variants. Buttons will automatically
adjust the color if it exists inside a dark background.

#### Contained

Contained buttons are high-emphasis, distinguished by their use of elevation and fill. They contain actions that are
primary to your app.

#### Outlined

Outlined buttons are used for more emphasis than Link buttons due to the stroke. Should be used in singular instances
within components.

#### Link

Used primarily in conjunction with a Contained button, however, when used alone, the button does not contain left
padding. This allows for additional flexible usage inside components.

## Size

Increasing the size will affect the Buttons' vertical padding, font size, and slightly increases line height.
